## 设计思路

1. **通用模板设计**：设计一个通用的POC模板，支持多种检测方式（关键字检测、时间盲注、状态码检测等）
2. **配置化**：通过JSON配置文件，支持批量检测多个目标
3. **多线程并发**：支持多线程并发，提高检测效率
4. **代理支持**：支持通过代理进行检测
5. **调试模式**：支持调试模式，输出详细的检测过程

## 使用方法

```bash
# 1. 安装依赖
pip install requests urllib3 pyyaml tqdm

# 2. 检测单个目标
python poc_engine.py -c poc_config.json -u http://192.168.1.100

# 3. 批量检测
python poc_engine.py -c poc_config.yaml -f urls.txt -o result.txt

# 4. 使用代理 + 调试模式
python poc_engine.py -c poc_config.json -f urls.txt --proxy http://127.0.0.1:7890 --debug
```

## 配置文件字段说明

| 字段              | 类型   | 必填 | 说明                                                         |
| ----------------- | ------ | ---- | ------------------------------------------------------------ |
| `name`            | string | 否   | 漏洞名称，用于日志输出                                       |
| `method`          | string | 是   | HTTP方法: GET/POST/PUT/DELETE                                |
| `path`            | string | 是   | 请求路径                                                     |
| `headers`         | object | 否   | 自定义请求头                                                 |
| `params`          | object | 否   | GET参数                                                      |
| `data`            | string | 否   | POST表单数据                                                 |
| `json_data`       | object | 否   | POST JSON数据                                                |
| `check_type`      | string | 是   | 检测方式: status_code/keyword/regex/time_based/header/not_keyword |
| `keyword`         | string | 否   | 关键字检测使用                                               |
| `expected_status` | int    | 否   | 状态码检测使用，默认200                                      |
| `sleep_seconds`   | int    | 否   | 时间盲注使用，默认5秒                                        |

## 支持这些POC的原因

从您的PDF中提取的POC，**90%以上**都可以通过这个通用模板覆盖：

| POC类型    | 特征                        | 通用模板支持方式          |
| ---------- | --------------------------- | ------------------------- |
| 关键字检测 | 响应中包含`XPATH`、`uid=`等 | `check_type: keyword`     |
| 时间盲注   | `SLEEP(5)`、`WAITFOR DELAY` | `check_type: time_based`  |
| 状态码检测 | 返回200即漏洞存在           | `check_type: status_code` |
| 文件上传   | 返回文件路径或成功状态      | `check_type: status_code` |
| 命令执行   | 响应中包含命令执行结果      | `check_type: keyword`     |

## 优势

1. **配置化**：新增POC只需添加一条JSON配置，无需写代码
2. **可维护**：POC集统一管理，便于更新和共享
3. **高效**：多线程并发，支持大批量目标
4. **安全**：支持代理和超时控制
5. **可扩展**：支持自定义检测逻辑
